[관리자] 주문 > 입금 전 관리

주문 선택 → 입금 확인 → 배송준비중 자동 전환

  • 취소된 주문은 [취소/교환/반품/환불] > 입금 전 취소 관리에서 확인
3미입금 자동 취소 설정
[쇼핑몰 설정] 주문 설정 > 주문 후 설정 > 미입금 주문 자동 취소
  • 사용함 선택 + 기한 설정 후 하단 저장

배송 준비중 관리
송장 입력 · 배송중 처리 · 묶음배송
▼
[관리자] 주문 > 배송 준비중 관리
1배송중 처리
  • 주문 선택 → 택배사 선택 + 송장번호 입력 → 체크 후 배송중처리 클릭
2배송대기 처리 (택배 연동용)
  • 택배사·송장번호 입력 → 송장번호 저장 → 배송 대기 관리로 이동
3묶음 배송 처리
  • 수령자 정보 동일한 주문 일괄 선택 → 묶음 배송 처리 클릭
  • 묶음배송 탭에서 택배사·송장 입력 후 배송중처리
4엑셀 일괄 처리
  • 페이지 하단 엑셀 양식 다운로드 → 수정 → 쇼핑몰·배송사 선택 → 파일 등록
⏳
배송 대기 관리
택배 연동 주문건 중간 처리
▼
[관리자] 주문 > 배송 대기 관리
1배송중 처리
  • 주문 체크 → 배송중 처리 → 배송중 관리로 이동
2배송준비중 되돌리기
  • 주문 선택 → 배송준비중 처리 → 배송준비중 관리로 이동
⚠️
연동택배 주문건은 임의로 상태를 변경하지 말 것. 배송대기까지 처리하면 1~2시간 내 택배 시스템에 전달되어 이후 실제 배송 상태에 따라 자동 전환됨.

2이전 단계로 되돌리기

배송완료 조회

▼
[관리자] 주문 > 배송 완료 조회
1조회
  • 원하는 조건 선택하여 검색
🚫
배송완료는 이전 상태(배송중)로 되돌릴 수 없음. 필요 시 사유·주문번호 기재하여 1:1 문의 접수 필요.
배송완료 자동체크 (유료 앱)
  • 택배 연동 미사용 쇼핑몰용 자동 처리 기능
  • 자동 처리 가능한 9개 택배사 주문 → 실제 배송 상황에 맞춰 자동 완료 처리
  • 설치: 앱 > 앱스토어 > 배송완료 자동체크 검색 후 설치 → 건수 충전 + 활성화

취소 관리
취소 접수 · 완료 · 철회
▼
[관리자] 주문 > 취소/교환/반품/환불 > 취소 관리
1고객 직접 취소신청 설정
[쇼핑몰 설정] 주문 설정 > 주문 정책 설정 > 주문 후 설정
  • 구매자 취소/교환/반품 신청 사용함
  • 신청 버튼 노출 범위 및 배송상태 체크 설정
2취소 접수
구매자 취소 신청 주문
  • 취소 관리 메뉴 → 취소처리 처리 버튼 → 상품 선택 + 취소 접수 → 구분 선택 후 확인
운영자 취소 처리
  • 전체주문조회 → 주문번호 클릭 → 주문 취소 → 상품 선택 + 취소 접수 → 구분 선택 후 확인
3취소 완료
  • 취소처리 중 상태는 환불처리까지 진행해야 완료 상태로 전환됨
  • 주문번호 클릭 → 환불 탭 → 환불처리 처리 버튼 → 환불완료처리
⚠️
무통장은 자동환불 불가 → 운영자가 직접 송금. 카드결제는 환불완료 처리 + PG 취소 처리 필요.
4취소 철회
  • 취소처리 상세 → 취소철회 → 구분·사유 기재 후 철회완료
🔄
교환 관리
배송 전/후 교환 · 차액 처리 · 철회
▼
[관리자] 주문 > 취소/교환/반품/환불 > 교환 관리
1배송 전 교환
유형처리 방법
금액 변동 없음접수 시 자동 교환 완료 → 교환 상품 포함 배송처리
차액 추가 결제고객 무통장 입금 → 입금전 관리에서 입금 확인 → 배송처리
차액 환불차액 환불 → 배송처리
2배송 후 교환
유형처리 방법
금액 변동 없음상품 회수 → 교환 관리에서 수거완료 → 교환 상품 배송
차액 결제 (왕복 배송비 포함)차액 입금 확인 → 상품 회수 + 수거완료 → 교환 상품 배송
차액 환불차액 환불 → 상품 회수 + 수거완료 → 교환 상품 배송
3교환 철회
  • 교환처리 상세 → 교환철회 → 구분 선택 후 철회완료
⚠️
금액 변동 있는 교환 철회 시 유의사항:
• 추가 입금 발생 → 고객에게 다시 입금 받은 후 철회
• 추가 환불 발생 → 고객에게 직접 환불 후 철회
• PG 취소 발생 → PG사 관리자페이지에서 철회 가능 여부 확인 후 진행
↩️
반품 관리
반품 신청 설정 · 접수 처리
▼
[관리자] 주문 > 취소/교환/반품/환불 > 반품 관리
1고객 직접 반품신청 설정
[쇼핑몰 설정] 주문 설정 > 주문 정책 설정 > 주문 후 설정
  • 구매자 취소/교환/반품 신청 설정 → 반품신청 항목에 배송 후 상태 체크
2반품 접수
구매자 반품 신청 주문
  • 반품 관리 → 반품처리 처리 버튼 → 상품 선택 + 반품접수 → 구분 선택 후 확인
운영자 반품 처리
  • 전체주문조회 → 주문번호 클릭 → 상품 반품 → 상품 선택 + 반품 접수 → 구분 선택 후 확인
💳
환불 관리
환불 접수 · 완료 · 철회
▼
[관리자] 주문 > 취소/교환/반품/환불 > 환불 관리
1환불 접수
  • 취소/교환/반품 접수 시 환불이 자동 접수됨
2환불 완료
환불처리 처리버튼 → 정보 입력 → 환불 완료 처리
💡
PG 결제 주문은 주문상세정보 창에서 PG 취소까지 완료해야 카드결제 승인 취소됨.
3환불 철회
  • 환불처리 상세 → 환불 전 처리 클릭 → 환불 전 상태로 변경
💴
결제수단별 환불 방법
카드 · 휴대폰 결제별 처리
▼
1카드 결제 주문
  • 취소 처리 후 주문상세정보 창 결제수단에서 신용카드 전체취소 / 신용카드 부분취소 클릭
⚠️
배송비 차감 또는 부분 취소/반품인 경우 부분취소 버튼 필수. 전체취소를 누르면 결제액 전액이 취소됨.
2휴대폰 결제 주문
  • 취소/반품 처리 후 결제수단에서 휴대폰결제 취소 클릭
  • 당월(1일~31일) 결제건만 취소 가능 (이통사 정책)
  • 결제월 지난 경우 적립금·예치금·무통장 등 다른 수단으로 환불
  • 부분취소 미지원 → 다른 수단 환불 또는 전체 취소 후 재주문 안내
📋
배송상태별 취소 방법
배송 전/후 부분·전체 취소 · 배송비 차감
▼
1배송 전 부분/전체 취소
전체주문조회 → 주문번호 클릭 → 주문 취소 → 상품 선택 + 취소접수
2배송 후 부분/전체 취소 (반품)
전체주문조회 → 주문번호 클릭 → 상품 반품 → 상품 선택 + 반품접수
💡
부분취소 ↔ 전체취소 처리 과정은 동일. 상품을 일부 선택하느냐 전체 선택하느냐의 차이.
3배송비 차감 환불 (고객 변심)
[쇼핑몰 설정] 주문 설정 > 주문 후 설정 > 입금/환불/반품처리 설정
동시 처리 설정인 경우
  • 취소/반품처리 팝업에서 상품 선택 + 접수 → 환불방식에서 반품배송비 적용 → 환불액 차감 금액 기재
분리 처리 설정인 경우
  • 취소/반품 접수 완료 후 → 주문상세정보 창 환불 탭 → 배송비·적립금·쿠폰 등 조정

예약주문 관리
예약/티켓 상품 관리 · 상태 변경
▼
[관리자] 주문 > 주문관리 부가기능 > 예약주문 관리
1예약 상품 유형
  • 숙박형 · 날짜+시간+사용자 옵션형 · 날짜+사용자 옵션형 · 사용자 옵션형
2예약 목록 조회
  • 리스트형: 여러 상태 주문 모아 조회 + 상태명 클릭 시 상태 변경 가능
  • 캘린더형: 주간/일간 예약 현황을 시간 단위로 확인
  • 예약상태 일괄 변경: 예약신청/예약대기/이용대기/이용완료 일괄 처리
📌
• 예약주문은 교환 불가 (취소·반품만 관리자에서 처리)
• 예약번호 = 쇼핑몰 어드민 주문번호 (품목 단위 확인)
• 배송정보 없음, 예약자 정보만 확인

구매확정 기능
설정 · 수동/자동 처리 · 취소
▼
1구매확정이란
  • 취소·교환·반품 의사 없음을 뜻함 → 확정 후 취소·교환·반품 불가
  • 배송중 또는 배송완료 상태일 때 구매자가 처리 가능
  • 불가피한 경우 운영자가 구매확정 취소 후 CS 처리
🚫
외부 주문(오픈마켓·네이버페이·카카오페이)은 어드민에서 구매확정으로 바꿔도 연동되지 않음.
2사용 설정
[쇼핑몰 설정] 주문 설정 > 주문 정책 설정 > 주문 후 설정 > 구매 확정 설정
  • 구매 확정 버튼 사용함 → 쇼핑몰 주문 조회 목록에 버튼 표시
3구매확정 처리
구매자
  • 마이페이지 주문 조회 목록의 구매 확정 버튼 클릭
운영자 - 수동
  • 주문번호 검색 → 주문상세정보 → 주문 상태 탭 → 상태 변경
운영자 - 자동
  • 구매 확정 자동체크 기능: 특정 기간 경과 시 자동 확정
  • 배송완료일 최근 1개월 이내 주문만 자동 처리 가능
4구매확정 취소
  • 주문상세정보 → 주문 상태 탭 → 구매 확정 취소 버튼 클릭
5구매확정 시 적립금 즉시 지급 설정
[쇼핑몰 설정] 고객 설정 > 회원 정책 설정 > 적립금 설정
  • 적립금 지급 시점 → 주문 상태 구매확정 후 + 지급 시점 즉시
🔢
할인금액 자동계산
CS 처리 시 할인액 자동 계산/회수
▼
[쇼핑몰 설정] 주문 설정 > 주문 정책 설정 > 주문 후 설정
1기능 개요
  • 주문 CS 처리 시 기존 할인금액을 자동 계산·적용
  • 반복적인 계산 업무 리소스 절감
  • 교환/반품으로 할인 조건 미충족 시 혜택 회수하여 손실 방지
  • 품목별 실 결제금액 제공으로 정산 오류 최소화
2설정 방법
  • 입금/환불/반품 처리 설정 → 취소/교환/반품 접수 시 품목별 할인/적립 금액설정 → 할인금액 자동 계산 선택
📌
설정 변경 시점부터 새로 생성된 주문에만 적용됨. 기존 주문은 소급 적용되지 않음.

네이버페이
가입 · 연동 · 주문 처리
▼
[관리자] 판매 채널 > 네이버 > 네이버 페이 안내
1가입 방법
서비스 신청 → 네이버페이 센터 → 신청서 작성 + 구비서류 → 1차 심사(약 1주)
⚠️
쇼핑몰에 교환/환불 불가 품목이 있으면 관련 법령 위반으로 네이버페이 가입 불가.
21차 승인 후 연동
[관리자] 판매 채널 > 네이버 > 네이버 페이 설정 > 서비스설정
  • 승인 메일의 가맹점 인증키·버튼인증키·네이버공통인증키 입력
  • 구매버튼 노출함, 주문연동 사용함 → 하단 등록
  • 이후 2차 심사(영업일 기준 평균 2일) → 최종 승인 후 이용 가능
3버튼 노출 설정
  • [상품] 상품목록 → 판매정보 탭 → 네이버 페이 서비스 사용함
  • [상점] 판매채널 > 네이버 > 네이버페이 구매 버튼 노출 노출함
  • [상품] 분류관리 > 상품 분류 관리 → 제휴서비스 사용함
  • [상품] 분류관리 > 기획전 분류 관리 → 제휴서비스 사용함
4버튼 미노출 사유
  • 판매가 0원·품절·대체문구 상품, 1+N 이벤트·해외배송 상품
  • 상품명/분류에 "개인결제·결제창·고객님" 등 포함된 상품
  • 쇼핑몰 로그인 여부 관련
5주문 처리
구분처리 방법
배송중 처리일반 주문과 동일 (주문관리는 네이버페이 센터 권장)
취소주문상세정보 → 주문취소 → 사유 입력
교환네이버페이 센터에서만 가능. 동일 상품만 교환, 옵션 변경 불가
반품구매자 요청 시 카페24에서 처리 가능. 승인 전 반드시 수거 확인
🚫
반드시 네이버페이 센터에서 처리:
• 반품 배송비 차감 등 부분환불
• 반품 철회 / 반품 거부
• 구매확정된 주문건
💡
일정 기간 배송 지연 처리 없이 진행되지 않으면 패널티 부과 가능. 재고 부족 시 네이버페이 센터에서 배송보류 처리 필요.

현금영수증
신청 · 발행 · 취소 · 의무발행
▼
[관리자] 부가서비스 > 기본 운영 서비스 > 현금영수증
1서비스 신청
  • 서비스 사용현황 → 신청하기 → 필수항목 기재
  • 신청서 사업자등록번호 = 쇼핑몰 정보의 사업자등록번호 일치 필수
⚠️
• 현금결제만 발급 가능 (계좌이체·가상계좌·무통장)
• 신용카드·휴대폰결제 발급 불가
• 발행금액에 배송비 포함, 적립금 사용액은 미포함
• KSNET 이용 시 자동 가입 안 됨 → 이니시스 현금영수증 별도 신청
2발행 설정
[쇼핑몰 설정] 결제 설정 > 매출 증빙 서비스 > 현금영수증 발행 설정
방식설명
자동발행 (권장)입금 완료·취소·반품 시 자동 처리 (고객이 신청해야 함)
수동발행운영자가 직접 발행·취소 처리 필요
3발급 방법
고객이 신청하지 않은 경우
  • 전체주문조회 → 주문번호 클릭 → 결제정보 탭 → 현금영수증 신청 버튼
4발급 내역 확인 / 취소
[관리자] 주문 > 현금영수증 관리 > 발행 내역 관리
  • 검색 → 발행건 확인, 미발행 건은 발행 버튼으로 처리
  • 취소: 발행취소 버튼 → 확인 → 발행취소 표시
  • 재발행: 발행하기의 재발행 클릭
PG사별 현금영수증 취소 가능 기간
PG사기간
KCP, 헥토파이낸셜(구 세틀뱅크)18개월
올앳, 이니시스(KSNET), 나이스페이, 토스12개월
스마트로3개월
5의무발행 설정
  • 발행 기능: 자동 또는 수동 발행
  • 의무 발행 설정: 사용함 + 신청금액 입력
  • 주문서 표시 설정: 표시함
💡
의무발행 사용 시, 고객이 원치 않아도 구매 금액이 총 신청금액 이상이면 국세청 지정번호(010-000-1234)로 자진 발급됨.

세금계산서
일반/전자 발행 · ETAX 연동
▼
[관리자] 주문 > 세금계산서 관리 > 발행 신청 관리
1발행 방법
일반 세금계산서
  • 발행 신청 관리에서 선택 → 확인 → 발행
  • 영수증 역할만 할 뿐 국세청 미전송 → 전자세금계산서 이용 또는 국세청 직접 발행 필요
전자 세금계산서
  • 고객 신청 + 배송완료 후 약 1시간 뒤 ETAX로 자동 전송
  • 취소·교환·환불 있으면 자동 전송 안 됨
  • ETAX 작성문서 관리 → 국세청 발행 버튼 클릭하여 국세청 전송
2매출 세금계산서 발행 (수동)
💡
고객이 미신청했거나, 인터넷 주문 외, 주문 내용과 다르게 발행 필요할 때 사용.
  • 홈 > 주요 부가서비스의 전자세금계산서 건수 클릭 → ETAX 이동
  • 문서 작성 → 매출 세금계산서 → 내용 입력 → 국세청 전송
3작성문서 관리 발행
구매자 미신청
  • 전체주문조회 → 주문번호 → 주문상세정보 → 세금계산서 신청 (배송완료 시만)
  • 사업자 정보 입력 → ETAX 문서관리 > 작성문서관리 → 국세청 전송
  • 범용 공인인증서 + 비밀번호 → 확인 → 상태 접수완료
구매자 신청
  • 홈 > 전자세금계산서 건수 → ETAX 작성문서관리 → 주문건 선택 → 국세청 전송
  • 범용 공인인증서 입력 → 확인 → 상태 접수완료
⚠️
유의사항
• 작성일 기준 다음 달 10일까지 발행 필수 (지연 시 가산세)
• 전자세금계산서는 공인인증서 전자서명 후 발행
• 발행일 지난 건은 관리자 발행 불가 → 국세청 e세로에서 직접 발행
• 가산세 관련은 국세청 문의
